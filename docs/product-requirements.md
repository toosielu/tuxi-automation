# 图喜 AI 小红书爆款内容生成系统 V1 功能文档

## 1. 产品一句话定位

图喜 AI 小红书爆款内容生成系统，是一个面向小红书虚拟电商用户的 AI 内容生成工具。

用户输入赛道、产品、人群和需求后，系统自动生成：

- 小红书爆款封面图方案
- 小红书爆款文案
- 商品承接文案
- 私信引导话术

核心目标是让用户每天快速生成可发布的小红书内容，用 AI 提升虚拟电商产品的交付价值和订阅收费能力。

## 2. V1 核心目标

V1 只做一个最小可用版本：

```text
用户输入产品信息
-> AI 生成爆款封面方案
-> AI 生成小红书文案
-> AI 生成承接话术
-> 用户复制使用
```

V1 不做复杂后台，不做自动发帖，不做爬虫，不做多账号管理。

## 3. 核心功能模块

### 3.1 首页输入模块

用户填写以下信息：

| 字段 | 类型 | 必填 |
| --- | --- | --- |
| 输入方式 | Enum | 是 |
| 原始文字素材 | Text | 否 |
| 图片内容描述 | Text | 否 |
| 参考链接 | String | 否 |
| 目标赛道 | String | 是 |
| 产品名称 | String | 是 |
| 目标人群 | String | 是 |
| 产品卖点 | Text | 是 |
| 内容目的 | Enum | 是 |
| 风格选择 | Enum | 否 |
| 当前卡点 | Text | 否 |

内容目的：

- 引流涨粉
- 私信咨询
- 店铺点击
- 产品成交
- 信任建立

风格选择：

- 干货型
- 结果型
- 避坑型
- 反差型
- 真实记录型
- 清单型

输入方式：

- 文字生帖子：粘贴资料、想法、产品介绍或参考结构
- 图片生帖子：上传参考图并补充图片内容描述，V1 先基于描述生成
- 链接生帖子：粘贴参考链接并补充想借鉴的角度，V1 先保存链接和补充说明

### 3.2 AI 爆款封面生成模块

系统根据用户输入，生成 3 个小红书封面方案。

每个封面方案包含：

| 字段 | 说明 |
| --- | --- |
| coverTitle | 封面主标题 |
| coverSubtitle | 封面副标题 |
| visualStyle | 视觉风格 |
| layoutSuggestion | 排版建议 |
| imagePrompt | AI 生图提示词 |
| reason | 为什么这个封面可能有点击率 |

封面要求：

- 标题短
- 重点突出
- 一眼能看懂
- 适合 1:1 小红书封面
- 避免夸张违规词
- 适合虚拟电商产品展示

### 3.3 AI 小红书文案生成模块

系统每次生成 3 条小红书帖子文案：

- 2 条流量爆款贴
- 1 条信任成交贴

每条文案包含：

| 字段 | 说明 |
| --- | --- |
| postType | 文案类型 |
| title | 小红书标题 |
| hook | 开头钩子 |
| content | 正文 |
| cta | 结尾引导 |
| tags | 推荐标签 |
| reason | 爆款逻辑说明 |

文案要求：

- 去 AI 腔
- 像真人经验分享
- 口语化
- 有具体场景
- 有收藏价值
- CTA 不能太硬
- 避免直接诱导违规导流

### 3.4 商品承接文案模块

系统根据产品信息生成商品承接内容。

输出内容：

| 字段 | 说明 |
| --- | --- |
| productTitle | 商品标题 |
| productSubtitle | 商品副标题 |
| sellingPoints | 商品卖点 |
| deliveryNote | 交付说明 |
| riskNotice | 风险提醒 |

用于：

- 小红书店铺
- 闲鱼商品页
- 微信小店
- 私域成交页

### 3.5 私信引导话术模块

系统生成 5 条低压私信回复话术。

场景：

- 初次咨询
- 价格咨询
- 犹豫观望
- 想要资料
- 成交引导

要求：

- 不强推
- 不油腻
- 不夸大收益
- 有信任感
- 适合个人 IP 私域成交

### 3.6 历史记录模块

保存用户生成过的内容。

保存字段：

| 字段 | 类型 |
| --- | --- |
| id | String |
| niche | String |
| productName | String |
| targetUser | String |
| generatedCovers | JSON |
| generatedPosts | JSON |
| createdAt | DateTime |

V1 先用内存存储，后续接 Supabase Postgres。

## 4. 页面结构

### 4.1 首页

页面内容：

- 产品标题
- 简短介绍
- 输入表单
- 一键生成按钮

核心按钮：生成爆款内容。

### 4.2 结果页

展示内容：

- 爆款封面方案
- 小红书文案
- 商品承接文案
- 私信引导话术

每个内容区提供复制按钮。

### 4.3 历史记录页

展示历史生成记录：

- 产品名称
- 赛道
- 生成时间

## 5. 后端接口设计

### 5.1 生成爆款内容接口

`POST /api/ai/generate`

请求参数：

```json
{
  "niche": "小红书虚拟资料",
  "productName": "小红书虚拟电商实操资料包",
  "targetUser": "想做副业的新手",
  "sellingPoints": "低成本、可复制、适合新手、自动发货",
  "goal": "产品成交",
  "style": "干货型",
  "painPoint": "不知道发什么内容，也不知道怎么承接成交"
}
```

返回参数：

```json
{
  "id": "",
  "covers": [],
  "posts": [],
  "productCopy": {},
  "dmScripts": [],
  "createdAt": ""
}
```

### 5.2 查询历史记录接口

`GET /api/history/list`

### 5.3 查询详情接口

`GET /api/history/detail/{id}`

## 7. AI 提示词要求

AI 生成时必须遵守：

- 输出 JSON
- 不要输出多余解释
- 文案要像真人写的
- 避免明显 AI 腔
- 避免夸大收益
- 避免违规引流词
- 适合小红书虚拟电商场景
- 强调真实经验、低成本、可执行、可复制

## 8. V1 不做功能

V1 暂不做：

- 自动发小红书
- 自动私信
- 自动爬取数据
- 登录注册
- 支付订阅
- 多账号矩阵
- CRM
- 团队权限
- 自动排程
- 复杂数据分析

第一优先级：

- 接 Supabase
- 接 AI 生图

## 11. 最终目标

V1 目标不是做复杂系统，而是先跑通一个最小闭环：

```text
用户输入产品
-> 系统生成爆款封面方案
-> 系统生成小红书文案
-> 系统生成成交承接话术
-> 用户复制发布
```

最终把小红书虚拟电商课程，从“卖资料”升级为：

```text
AI 爆款内容生成系统 + 陪跑服务 + 订阅收费
```
