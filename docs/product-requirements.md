# 图喜小红书虚拟自动化成交系统产品功能文档

## 1. 产品定位

图喜小红书虚拟自动化成交系统是一套面向虚拟电商项目的自动化运营工作台，目标是帮助用户从赛道判断、产品匹配、内容生成、发布排程到数据复盘，形成可复制的成交链路。

系统不是课程介绍页，也不是单次文案生成器，而是围绕「帖子 -> 流量 -> 店铺 -> 出单 -> 私域」持续推进的自动化系统。

## 2. 目标用户

### 2.1 完全新手

用户特征：

- 想做小红书虚拟电商，但不知道做什么赛道
- 没有稳定产品库
- 不会写帖子和店铺转化文案
- 希望低成本测试首单

核心诉求：

- 找到可做赛道
- 匹配适合自己的虚拟产品
- 获得可直接执行的发帖和上架方案

### 2.2 已有账号或店铺的人

用户特征：

- 已经发过内容或上架过产品
- 有曝光但咨询少，或有咨询但成交少
- 缺少内容复盘和私域承接 SOP

核心诉求：

- 找出成交链路卡点
- 优化内容、店铺、私域承接
- 提升出单稳定性

## 3. V1 核心链路

V1 先跑通以下链路：

```text
创建项目
-> 自动筛选赛道
-> 自动匹配产品库
-> AI 生成小红书帖子
-> 生成图片提示词
-> 加入发布排程
-> 手动录入数据复盘
-> 输出下一轮优化建议
```

V1 不做真实小红书全自动发帖。发布动作先做排程、复制和人工确认，避免账号风控、验证码和平台规则变化导致系统不可用。

## 4. 功能模块

### 4.1 项目管理

用户可以创建一个自动化项目。

字段：

- 项目名称
- 目标赛道
- 目标人群
- 当前阶段：新手起盘、已有账号、已有店铺
- 每日可投入时间
- 启动预算
- 转化目标：店铺出单、私域咨询、资料包成交、AI 批量提效
- 当前卡点

输出：

- 项目 ID
- 项目基础信息
- 创建时间
- 后续所有分析和内容生成记录都归属到该项目

### 4.2 自动筛选赛道

系统根据项目输入生成赛道分析。

评分维度：

- 需求强度
- 变现适配度
- 执行难度
- 预算适配度
- 合规安全度

输出：

- 综合评分
- 推荐等级：优先测试、可以小规模验证、暂缓
- 赛道总结
- 风险提示
- 下一步建议

### 4.3 产品库管理

产品库存储在 Supabase Postgres。

产品字段：

- 产品名称
- 产品分类
- 适合人群
- 产品类型：资料包、模板、产品库、话术包、工作流
- 定价区间
- 交付方式
- 执行难度
- 风险点
- 卖点列表

后台能力：

- 新增产品
- 编辑产品
- 删除产品
- 查询产品列表
- 根据赛道和人群匹配产品

### 4.4 自动匹配产品

系统根据项目的赛道、人群、阶段、目标自动匹配产品库。

输出：

- 推荐产品列表
- 每个产品的匹配分
- 推荐原因
- 定价建议
- 上架标题
- 详情页卖点
- 自动交付建议
- 合规风险提醒

### 4.5 AI 生成小红书帖子

系统根据赛道和推荐产品生成小红书内容包。

每条帖子包含：

- 内容角度
- 爆款标题
- 封面文案
- 正文
- CTA
- 评论区引导话术
- 私信引导话术

V1 生成 5 条内容，后续扩展到批量 10-30 条。

### 4.6 图片生成提示词

V1 先生成封面图提示词，不直接生成图片。

输出：

- 封面图 prompt
- 信息图 prompt
- 产品展示图 prompt

后续可以接入图片生成模型，并把结果存入 Supabase Storage。

### 4.7 发布排程

用户可以把生成的帖子加入发布计划。

字段：

- 帖子 ID
- 平台：小红书
- 计划发布时间
- 发布状态：待发布、已发布、已复盘
- 已发布链接
- 备注

V1 支持人工发布记录。V2 再做浏览器自动化辅助发布。

### 4.8 数据复盘

用户手动录入发布后的笔记表现。

字段：

- 曝光
- 点赞
- 收藏
- 评论
- 私信数
- 店铺点击
- 成交数

系统计算：

- 收藏率
- 私信率
- 成交率

输出：

- 内容问题判断
- 产品承接问题判断
- 下一轮选题建议

## 5. 数据存储方案

### 5.1 Supabase Postgres

存储结构化业务数据：

- 用户
- 项目
- 赛道分析
- 产品库
- 生成帖子
- 发布排程
- 复盘数据
- AI 生成日志

### 5.2 Supabase Storage

存储文件类资产：

- 小红书封面图
- 产品展示图
- 交付资料附件
- 生成素材

数据库只保存文件 URL 和元数据，不直接存图片二进制。

## 6. 最小数据库表

### 6.1 projects

保存自动化项目。

### 6.2 niche_analysis

保存赛道分析结果。

### 6.3 products

保存虚拟产品库。

### 6.4 generated_posts

保存 AI 生成的小红书帖子。

### 6.5 publish_schedules

保存发布排程。

### 6.6 post_metrics

保存发布后的数据复盘。

### 6.7 ai_generation_logs

保存 AI 请求和生成结果，便于后续排查和优化。

## 7. 技术架构

### 7.1 前端

V1 使用原生 HTML、CSS、JavaScript，继续复用当前工作台页面。

后续可以迁移到 React / Vue / Next.js，但 V1 不增加复杂度。

### 7.2 后端

后端迁移为 Java Spring Boot。

核心模块：

- Project API
- Niche Analysis API
- Product API
- Post Generation API
- Publish Schedule API
- Metrics API

### 7.3 数据库

使用 Supabase Postgres。

Java 后端通过 JDBC / MyBatis 访问数据库。前端不直接访问 Supabase，统一通过后端 API 读写，便于权限控制和后续商业化。

## 8. V1 开发顺序

1. 创建 Spring Boot 后端骨架
2. 配置 Supabase 数据库连接
3. 建立 SQL schema
4. 实现产品库读取
5. 实现项目创建
6. 实现赛道分析服务
7. 实现产品匹配服务
8. 实现帖子生成服务
9. 实现发布排程保存
10. 前端切换到 Java API
11. 本地 Git 管理
12. GitHub 插件准备好后接入远程仓库

## 9. V1 暂不做

- 真实小红书一键发帖
- 多账号自动登录
- 验证码处理
- 支付订阅
- 团队协作
- 复杂权限
- 向量数据库
- 图片直接生成

这些功能放到 V2/V3，避免第一版过重。
